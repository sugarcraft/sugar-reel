# Caliber Learnings

Accumulated patterns and anti-patterns from development sessions.
Auto-managed by [caliber](https://github.com/caliber-ai-org/ai-setup) — do not edit manually.

- **[fix]** When a merged PR leaves a stale `dev-ai/<old-branch>` pin on `sugarcraft/<dep>` inside a consuming lib's `composer.lock` — surfacing post-merge as CI "Interface SugarCraft\…\X not found" / class-load errors — fix on a NEW branch off master: (1) `cd <lib> && composer update sugarcraft/<dep>` to regenerate, (2) `sed -i 's|dev-ai/<new-branch>|dev-master|g' <lib>/composer.lock` to rewrite composer's new pin back to `dev-master`, (3) `composer validate` + full PHPUnit run, (4) commit & PR. Composer's `update` will always pin to the current branch name; the `sed` step is what makes the lock match the canonical `dev-master` shape the rest of the monorepo uses. IF that PR's master CI still fails identically, the `restore-keys` cache shadow is the next layer — bump cache-key version (see cache-bump gotcha below).
- **[pattern]** Two-step CI lock+cache fix recipe: when a `dev-ai/<old-branch>` pin escapes to master, the FIRST PR fixes the lockfile (see lock-pin fix above), and if its post-merge master CI run STILL fails with the same class-not-found, ship a SECOND immediate PR that bumps the cache-key `vN` suffix across all keys. Don't try to fix both in one PR — the lock-pin PR has a tiny diff that's easy to review, and the cache-bump PR is mechanical sed. Canonical sequence: PR #487 (lock pin) → PR #488 (hotfix lock pin again) → PR #489 (cache bump v2→v3) → master green on run 25986272229.
- **[pattern:vhs-runner-php]** `ghcr.io/detain/sugarcraft-vhs-runner-php:latest` is the PHP-based VHS runner image built from `scripts/Dockerfile.vhs-runner` (php:8.3-cli-bookworm + ffmpeg + gd + mbstring + intl + curl + ssh2 + JetBrainsMono fonts from candy-vcr/fonts/). Used by the `vhs-candy-vcr` job in vhs.yml for A/B parallel rendering alongside the Go-based `vhs-runner` image. After a 2-week soak the PHP runner replaces the Go runner. Local build: `docker build -f scripts/Dockerfile.vhs-runner -t sugarcraft-vhs-runner-php:local .`
- **[pattern:mutable-trait]** Standard immutable-with pattern via `SugarCraft\Core\Concerns\Mutable` trait: `use Mutable;` in the class, then `mutate(array $changes): static { return new static(...array_merge(get_object_vars($this), $changes)); }`. Works for simple cases where all properties are set via constructor and mutable fields are private. **Exception:** Libs with sentinel-bool needs (Style's `with()` method tracks `propsSet`; TextArea's `mutate()` has `validateSet`/`errSet`/`promptFuncSet` sentinels) must override `mutate()` in-class and document the exception. See `candy-core/src/Concerns/Mutable.php`, `candy-sprinkles/CALIBER_LEARNINGS.md` (style-with exception), `sugar-bits/CALIBER_LEARNINGS.md` (sentinel-bool pattern).
- **[convention]** When doing a byte-identical SGR→`Ansi` refactor, `\033` (octal) and `\x1b` (hex) are the SAME ESC byte (0x1B): `Ansi::sgr(1,33)` is byte-identical to both `"\033[1;33m"` and `"\x1b[1;33m"`, `Ansi::reset()` === `"\x1b[0m"`, `Ansi::CSI` === `"\x1b["`. So routing inline literals through `Ansi::sgr()`/`reset()`/`CSI` does NOT change emitted bytes and snapshot tests stay green. Only `…m`-terminated sequences are SGR — `\033[2J`/`\033[H`/`\033[?25l` (erase/home/cursor) are NOT, leave them; `\033[49m` (default-bg) IS SGR, convert it. Verify nothing remains with `grep -rnE '"\\x1b\[[0-9;]*m|\\033\[[0-9;]*m' <lib>/src | grep -v 'Ansi::'` (empty = clean).
- **[convention]** Root `improvements.md` is the monorepo three-axis audit doc (§1, §3.3 SGR cleanup, §3.4 Boxer, …). Mark completed items ✅ inline where they live and cite the PR number (`✅ Batch B cleaned (PR #881): …`). Commit the change with a placeholder `(this PR)` in the same branch, then a follow-up commit rewrites `(this PR)` → `(PR #<n>)` once the number is known. Same in-place history convention as `plans/AUDIT_*.md`.
- **[pattern]** `gh pr merge <n>` can fail right after `gh pr create` because GitHub hasn't finished computing mergeability — poll `gh pr view <n> --json state,mergeable -q '"\(.state)/\(.mergeable)"'` in a short retry loop (sleep 8 between attempts), merge only when it returns `*/MERGEABLE`, break on `MERGED*`. Prefix the whole loop with `unset GITHUB_TOKEN`. Avoids a spurious not-mergeable failure on a freshly-pushed branch.
- **[convention]** `sugar-readline` deliberately has NO candy-core dependency — its only SGR lives in a single self-contained `sugar-readline/src/Ansi.php` helper. Do NOT route it through `SugarCraft\Core\Util\Ansi` or add a candy-core dep during SGR-cleanup sweeps; it is intentionally excluded.
- **[gotcha]** `fprintf($stream, $sprintfBuiltString)` is a trap when the second arg was built via `sprintf` and contains literal `%` (e.g. left over from `%%`): `fprintf` re-interprets the percent signs as format specifiers and throws `ArgumentCountError`. Use `fwrite($stream, $alreadyFormattedString)` for any already-substituted string. (SGR escapes never contain `%`, so `Ansi::reset()`/`Ansi::sgr()` are safe to leave inside a sprintf format string.)
- **[gotcha]** Real-PTY tests MUST call `\stream_set_blocking($pair->master()->stream(), false)` before any read on the master — otherwise `fread` ignores the `$timeout` arg and blocks the whole PHP process until the kernel buffer has data. Symptom: a test that should raise `ExpectTimeoutException` hangs forever.
- **[gotcha]** macOS arm64 + PHP FFI + variadic libc functions = ABI mismatch. Real `ioctl(int, unsigned long, ...)` is variadic; a fixed-arg FFI cdef pushes the third arg into `x2` while the kernel reads it from the stack → ioctl returns `-1` with no useful errno. On Darwin route through non-variadic `tcsetwinsize`/`tcgetwinsize` or shell out to `stty -f /dev/fd/<fd>`.
- **[gotcha]** Bash watchdog `( sleep N && pkill ... ) &` placed before/around a `cd /path/to/lib` loses CWD — even `cd /lib && ( sleep N && pkill ) >log 2>&1 &` backgrounds the WHOLE compound (cd included) so the foreground phpunit runs in the parent CWD with exit 127. SAFEST: use the lib's ABSOLUTE phpunit path (`/home/sites/sugarcraft/<lib>/vendor/bin/phpunit`) and anchor every dependent path (cassette tmp, --bootstrap, --configuration) absolutely.
- **[pattern]** Cleaner watchdog cleanup: capture `WD=$!` right after the backgrounded `( sleep N && pkill -f phpunit ) &`, run the foreground command, then `kill $WD 2>/dev/null` BEFORE inspecting output — otherwise the watchdog survives and may kill an unrelated phpunit N seconds later in the same shell session.
- **[gotcha]** Tracking "has this float been initialized?" with `$prevField > 0.0` is fragile when the field is `round(microtime(true) - $startTime, 3)` — the first event in the first millisecond legitimately produces 0.0, so the guard mistakes it for "never set". Use a separate `bool $hasPriorEvent = false` flag. Canonical: `candy-vcr/src/Recorder.php::writeEvent()`.
- **[pattern]** Host TTY safety-net for any CLI that drops host stdin into raw mode: `register_shutdown_function` + `pcntl_signal(SIGTERM/SIGHUP)` with `pcntl_async_signals(true)`, hold the Termios snapshot in a STATIC class property (closures can't survive past `run()`), drop a `sys_get_temp_dir().'/<cmd>-rescue.<pid>'` marker with `posix_ttyname(STDIN)` so external `stty sane < /path/to/tty` recovers from SIGKILL. Handlers must be signal-safe + idempotent. Canonical: `candy-vcr/src/Cli/RecordCommand.php`.
- **[gotcha]** Reset ALL static properties in `tearDown()` via Reflection when testing static-property class state — PHPUnit test order is not source order, and a leaked snapshot from one test makes the next test's "no snapshot installed" assertion fail mysteriously.
- **[pattern]** PTY-real PHPUnit tests skip cleanly on FFI-less/sandboxed CI via a `requirePtySyscalls()` preamble: check `PHP_OS_FAMILY !== 'Windows'`, `extension_loaded('ffi')`, `is_readable('/dev/ptmx') && is_writable('/dev/ptmx')`, `extension_loaded('pcntl')` (when `controllingTerminal: true`), and `is_executable` of the child binary. For interactive children (vim) also probe `command -v vim` as a fallback to a hard-coded path.
- **[gotcha]** Real-PTY tests spawning a child with `sleep N` between outputs MUST give `RecordCommand` a non-EOFing stdin — `\fopen('/dev/null','r')` EOFs immediately and `PumpOptions::stdinEofGraceSec` closes the master before the child wakes. Use `\stream_socket_pair(...)` and keep BOTH ends open for the test duration, close both in `finally`.
- **[convention]** `candy-vt`'s `SugarCraft\Vt\Screen\Screen` exposes dimensions as PUBLIC READONLY PROPERTIES — `$screen->cols`/`$screen->rows`, NOT methods. `$screen->cols()` errors with `Call to undefined method`. Same on `Buffer` (`$buf->cols`/`$buf->rows`). Breaks the bare-named-accessor mental model used elsewhere.
- **[pattern]** Recorder/pump overhead microbench: one untimed warmup (seeds FFI cdef + kernel pty allocator + autoload caches), then N=5 timed iterations/scenario, sort, take the MEDIAN (robust vs single-shot GC/preemption stalls), assert `(median_with - median_without)/median_without ≤ budget`. Local Linux noise floor is ~0% (negative is normal); set CI budget 5% to absorb shared-runner jitter. Canonical: `candy-vcr/tests/Integration/ShirleyOverheadTest.php`.
- **[fix]** A stale `dev-ai/<old-branch>` pin on `sugarcraft/<dep>` in a consuming lib's `composer.lock` surfaces post-merge as CI `Interface SugarCraft\…\X not found`. Fix on a NEW branch off master: (1) `cd <lib> && composer update sugarcraft/<dep>`, (2) `sed -i 's|dev-ai/<new-branch>|dev-master|g' <lib>/composer.lock`, (3) `composer validate` + full phpunit, (4) commit & PR. `composer update` always pins to the current branch name; the `sed` rewrites it to the canonical `dev-master`. Confirm locally with `composer install --dry-run | head -20` (a clean master shows `Nothing to install`).
- **[gotcha]** A class-not-found in CI on a freshly-merged-to-master commit (passing locally) is almost always EITHER a stale `composer.lock` cross-lib pin OR a stale `actions/cache@v5` lockfile shadow. Checks: (1) `grep -A1 '"name": "sugarcraft/<dep>"' <lib>/composer.lock` — `dev-ai/<old>` not `dev-master` = lock pin; (2) if the lock is correct, a `restore-keys:` partial-match cache restore OVERWRITES the committed lock before `composer install` — bump the cache-key `vN` suffix to fix.
- **[convention]** Cache-key version bump in `.github/workflows/ci.yml`: bump EVERY `-vN-` key in lockstep (`lock-cov-vN`, `vendor-vN`, `vendor-cov-vN`, `vendor-stan-vN`, `vendor-win-vN`, `vendor-mac-vN`). Enumerate with `grep -n 'cov-v\|vendor-v\|key: lock-'` then `sed` in one pass — partial bumps leave shadowed vendor/ trees re-resolving to the stale lock.
- **[pattern]** Two-step CI lock+cache fix: the FIRST PR fixes the lockfile pin (tiny reviewable diff); if its post-merge master CI STILL fails identically, ship a SECOND mechanical PR bumping the cache-key `vN` suffix. Don't bundle them.
- **[gotcha]** `gh run view <id> --log-failed | grep -E '(FAIL|Error|Tests:)'` often returns empty even on a loud failure — log lines are prefixed `<job>\t<step>\t<ts>` and PHPUnit fatals don't carry those tokens. Use `grep -B1 -A6 'inside PHPUnit'` or fall back to `--log-failed 2>&1 | tail -80`.
- **[gotcha]** `gh run list --workflow=ci.yml --branch=master --limit=1` returns the MOST RECENT run regardless of state — a fresh merge shows `queued`/`in_progress` with empty `conclusion`. Use `--status=success --limit=1` to confirm CI is actually green.
- **[pattern]** Summarize a CI run cleanly: `gh run view <id> --json conclusion,jobs | python3 -c "import sys,json; d=json.load(sys.stdin); jobs=d['jobs']; fail=[j['name'] for j in jobs if j['conclusion']=='failure']; print('overall',d['conclusion'],len(jobs)-len(fail),'pass /',len(fail),'fail'); [print(' FAIL:',n) for n in fail]"` — faster than scrolling `--log-failed`.

Libs with sentinel-bool needs (Style's `propsSet`, TextArea's `validateSet`/`errSet`/`promptFuncSet`) must override `mutate()` in-class and document the exception.

- **[pattern]** Driving vim in a deterministic test: spawn with `-N -u NONE -i NONE <scratch>`, pre-load the entire keystroke script onto a `stream_socket_pair()` write-end BEFORE `RecordCommand::run()`, keep BOTH ends open until run() returns. Accept exit code 0 OR 1 (`:wq` returns 1 on some builds when the buffer is unchanged). Canonical: `candy-vcr/tests/Integration/ShirleyVimTest.php`.
