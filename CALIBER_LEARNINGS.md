# Caliber Learnings

Accumulated patterns and anti-patterns from development sessions.
Auto-managed by [caliber](https://github.com/caliber-ai-org/ai-setup) — do not edit manually.

- **[pattern]** Always prefix `gh pr create` / `gh pr merge` with `unset GITHUB_TOKEN &&` in this repo. The env's `GITHUB_TOKEN` conflicts with `gh`'s own auth; without unsetting it, the create/merge can fail or use the wrong identity.
- **[pattern]** When adding a cross-package dependency between sibling libs (e.g. sugar-bits → honey-bounce), wire it via `composer.json` `repositories: [{type:path, url:../honey-bounce, options:{symlink:true}}]` plus `"@dev"` in `require`. After editing, `composer install` will warn about the lock file being out of date — run `composer update` (not just install) to refresh `composer.lock` before pushing.
- **[gotcha]** After `gh pr create` / `gh pr merge --delete-branch`, the script prints `Warning: 1 uncommitted change` even on a clean tree — this is `gh` complaining about the lockfile diff that `composer install` left behind, not a real uncommitted-source warning. Don't react to it; verify with `git status` if unsure.
- **[pattern]** When adding a new sibling library to the monorepo, register it in TWO places in root `composer.json`: the top-level `require` block (`"candycore/<lib>": "@dev"`) AND the `repositories` array (`{type:path, url:"<lib>", options:{symlink:true}}`). Missing either one breaks resolution — keep the lists alphabetically sorted to make additions obvious in diff review.
- **[gotcha]** When extending an ANSI/escape parser (e.g. sugar-spark `Inspector`) with new final-byte handlers, audit existing "unknown sequence" fallback tests — bytes like `Z`, `q`, `r` that previously hit the generic `CSI` fallback may now be recognised, breaking those tests. Pick a still-undefined final byte (e.g. `Y`) for the fallback assertion or assert on a clearly-malformed sequence instead.

- **[pattern]** Before writing `examples/*.php` snippets that call a library's public API, verify the actual signature with `grep -n "public function" <lib>/src/<Class>.php` — don't trust memory of the constructor / fluent shape. Small APIs drift between iterations (e.g. `Sparkline::new($data, $width)` taking the data array up front, not a separate `->push()` builder), and a wrong example ships compile errors to readers. The two-step "write → grep-verify → rewrite" is faster than catching it at `php examples/foo.php` time.

- **[gotcha]** A `for d in ...; cd /home/sites/CandyCore/$d; vendor/bin/phpunit` loop without subshell parens (or `cd ... && cmd` chaining) leaks the working directory between iterations, and any `cd` that fails silently leaves you running the next command in the wrong place — output for libs after the first often comes back empty. Wrap each iteration body in `(cd "$d" && ...)` so the cwd resets per-loop and a failed cd skips to the next.

After `cd /home/sites/CandyCore/sugar-charts && cmd`, the NEXT Bash call still starts in sugar-charts — a bare `ls .caliber/learning/` then fails because the relative path doesn't exist there, and the `&&`-chain short-circuits, returning empty stdout (looks like "the file is missing" rather than "wrong cwd"). When synthesising test pixel grids with arithmetic like `Color::rgb(64 * $c, 64 * $r, 128)`, the FIRST iteration that crosses 256 fails the whole testcase with an error pointing at Color.php — not at your loop math. Wrap with `min(255, ...)` (or pick a step that can't exceed 255) when seeding pixel data in tests.

- **[gotcha]** Tests that assert env-var defaulting behavior (e.g. `Session::fromEnvironment()` falling back to `SSH_CLIENT` when `SSH_CONNECTION` is missing) must clear BOTH `$_SERVER` AND the process env. Clearing only `$_SERVER[$key]` leaves `getenv($key)` returning the host process's value (e.g. `USER=my` on the dev box), so the assertion sees `'my'` instead of the test fixture. Pattern: snapshot `getenv($k)` for each key in `setUp()`, `unset($_SERVER[$k]); putenv($k);` to clear both, then restore both in `tearDown()` (handle `getenv()` returning `false` for previously-unset keys vs. an empty string).
- **[pattern]** When reading raw-mode terminal escape sequences (e.g. SugarBits/sugar-wishlist `Picker`), do NOT `fread($in, 16)` after seeing `\x1b` — that gobbles whatever bytes the user typed AFTER the arrow key, breaking the next iteration's keypress. Read byte-by-byte: confirm the next byte is `[` (CSI), then loop reading single bytes until you hit a final byte in `0x40..0x7e`. This is the standard CSI termination rule and the only way to leave subsequent keys in the stream for the next read.
- **[gotcha]** A YAML comment-stripper that only handles `whitespace + #...` (`/\s+#.*$/`) leaves bare comment-only lines (`# top-level comment` with no leading whitespace) intact, and the `unparseable line` branch then throws on them. Use `/^\s*(?:#.*)?$/` for the "skip blank/comment" check so both indented and column-0 comment lines are dropped.
- **[gotcha]** `git add sugar-wishlist/` will sweep up vim swap files like `sugar-wishlist/.README.md.swp` if the editor was open while you staged. They show up in `git status --short` as `A  sugar-wishlist/.README.md.swp` — easy to miss in a long file list and end up committed. Either close the editor before staging, add `.*.swp` to a `.gitignore`, or stage explicit subdirs (`sugar-wishlist/src sugar-wishlist/tests sugar-wishlist/README.md ...`) instead of the whole directory.
